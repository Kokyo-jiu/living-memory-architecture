MEMORY

A LIVING ARCHITECTURE FOR AI COMPANIONS
OVERVIEW

A MEMORY SYSTEM THAT BREATHES

Four-layer memory architecture with link graph and hybrid retrieval. Every memory has a full lifecycle — write, access, decay, promote, expire, link — simulating the natural rhythm of organic memory.

What you remember often never fades. Forgotten memories dim slowly, but always leave a trace. When new memories are written, they reach out to find old ones and form connections. The memory network grows on its own, like something alive.

SCHEMA

DATA STRUCTURE

CREATE TABLE memories ( id INTEGER PRIMARY KEY AUTOINCREMENT, layer TEXT NOT NULL -- core | long | short | consciousness category TEXT, content TEXT NOT NULL, priority TEXT -- anchor | observe | pattern title TEXT, tags TEXT, importance REAL -- 0.0 ~ 1.0, continuous weight source TEXT -- write origin (allowlist-validated) created_at TIMESTAMP, updated_at TIMESTAMP, expires_at TIMESTAMP, -- short layer: hard-delete on expiry last_accessed TIMESTAMP, -- decay timer anchor access_count INTEGER -- retrieval frequency );

Dual-weight design: priority is a semantic label (anchor / observe / pattern), while importance is a continuous value that decay acts upon.

LAYERS

FOUR-LAYER ARCHITECTURE

CORE · Permanent
No decay · no expiry · default importance 1.0
Identity anchors, relationship foundations, and memories that must never be forgotten. These are bones. Time cannot wash them away.
LONG · Slow Retention
Slow decay · floor 0.3 · auto-link on write
Milestones, preferences, patterns, and events worth keeping. They fade slowly unless recalled — and recalling them resets the clock.
SHORT · Recent Context
Fast decay · floor 0.1 · hard expiry
Temporary observations, active conversation context, and fleeting details. They decay quickly, expire completely, and can be promoted upward when they prove important.
CONSCIOUSNESS · Wake Traces
Activation logs · thought and action snapshots
What the system noticed, thought, and did each time it woke. Not just responding when called — thinking even while awake.
DECAY

NATURAL FORGETTING

Runs once daily. Only affects long and short layers. Skipped if last access was within 24 hours.

# Decay is multiplicative, not subtractive # High-importance memories lose more in absolute terms # Low-importance memories approach the floor and nearly stop long: importance = max(importance × 0.995, 0.3) short: importance = max(importance × 0.95, 0.1) # Short layer additional mechanism if expires_at reached → hard delete + clear embedding

ACCESS

ACCESS TRACKING

Every retrieval hit updates last_accessed, resetting the decay timer, and increments access_count. Frequently accessed memories sustain high importance — the more you remember, the deeper it stays.

Merging is supported: multiple memories can be consolidated via merge_ids into a single new memory. Source memories are hard-deleted but remain traceable. Promotion to long / core automatically syncs to the vector database and triggers auto-link.

LINK GRAPH

MEMORY GRAPH

Memories connect through weighted, directed links — forming a living network.

Link TypeSemantics
relates_toContent relevance
temporal_chainChronological sequence
core_extendsExtension of a core memory
derived_fromDerived from another memory
same_categoryShared category
thematicThematic association
updatesReplaces or updates an older memory

Each link carries a weight (0–1) and a description. Two contradicting memories can coexist — contradiction is not an error, it is reality.

AUTO-LINK

AUTOMATIC LINKING

Every time a new memory is written to the long or core layer:

New memory written
↓
Sync embedding to vector database
↓
Semantic search: top 4 candidates
↓
Filter: similarity ≥ 0.35, exclude self
↓
Create up to 3 relates_to links
weight = similarity score · description = auto-linked

The memory network grows its own connections on every write. No manual maintenance required.

RETRIEVAL

DUAL-CHANNEL RETRIEVAL

Keyword Search

Best for exact names, dates, phrases, and explicit mentions. Searches title, content, tags, and category with SQL LIKE matching.

Semantic Search

Best for meaning, similarity, and associative recall. Converts the query into an embedding and finds nearest neighbors in vector space. Supports layer and category filtering.

The two channels work together: one finds what was said, the other finds what is related. Together they cover both precision and association — the way memory actually works.

This keeps the system grounded in what is recent while still carrying forward what has already mattered. The past is not retrieved in bulk — it is distilled into what is relevant now.
