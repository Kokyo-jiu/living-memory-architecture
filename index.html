A Living Memory Architecture

This project explores how an AI companion can maintain persistent, evolving, and context-aware memory across conversations. The goal is not to store everything, but to remember what matters, forget gently, and reconnect meaning over time.

Layered persistence Natural forgetting Access-based retention Weighted link graph Hybrid retrieval Context injection

Core Idea

The system treats memory as something alive: important memories stay stable, recent memories remain fluid, unused memories slowly fade, and related memories connect to each other over time.

Instead of isolating notes, the architecture keeps memory as a network of meaning, continuity, and change.

Four Memory Layers

COREpermanent anchors

Identity, relationship anchors, and the most stable memory points. These do not decay.

LONGslow retention

Milestones, preferences, patterns, and important events. They fade slowly unless recalled.

SHORTrecent context

Temporary observations and active conversation context. These decay quickly and can be promoted.

CONSCIOUSNESSwake traces

Activation snapshots and wake logs that record what the system noticed, did, and carried forward.

Write New memory enters the system with layer, category, importance, and source.
Retrieve Keyword and semantic search work together to find relevant memories.
Access Update Frequently accessed memories refresh their last-accessed timestamp.
Decay Unused memories gradually lose importance instead of staying frozen forever.

Example Decay Model

One simple configuration looks like this:

long_memory.importance = max(importance × 0.995, 0.3) short_memory.importance = max(importance × 0.95, 0.1) if recently_accessed: skip_decay()

Memories that are recalled often remain strong; memories that are not recalled slowly fade.

Link Graph

Memory is not stored as isolated notes. Each memory can connect to other memories through typed, weighted links.

Link Types

  • relates_to
  • same_category
  • temporal_chain
  • derived_from
  • updates
  • contradicts
  • thematic
  • core_extends

Why it matters

Contradiction is preserved instead of erased, because real memory can contain tension. Meaning does not stay in one note; it spreads through connections.

Retrieval

Keyword Search

Best for exact phrases, names, dates, and explicit mentions.

Semantic Search

Best for meaning, similarity, and related context.

The two channels work together: one finds what was said, the other finds what is related.

Context Injection

Before each response, the system builds a compact memory context from recent traces and long-term relevant memories.

Input sources

  • Recent observations
  • Wake logs
  • Conversation summaries
  • Time-sensitive diary entries
  • Semantic memory matches

Purpose

This lets the system stay grounded in what is recent while still carrying forward what has already mattered.
